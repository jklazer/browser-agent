# AI Browser Agent

Автономный AI-агент, управляющий реальным браузером для выполнения сложных многошаговых задач.

Пользователь пишет задачу текстом в терминале — агент сам открывает сайты, находит элементы, заполняет формы, кликает и возвращает результат.

## Как работает

```
Пользователь           Agent (Claude CLI)         Playwright (Chromium)
    │                       │                           │
    │── "Найди вакансии" ──>│                           │
    │                       │── navigate(hh.ru) ──────>│
    │                       │<── page state ───────────│
    │                       │── type_text("AI инж") ──>│
    │                       │── press_key("Enter") ───>│
    │                       │── get_page_state ───────>│
    │                       │<── результаты ───────────│
    │<── 3 вакансии ────────│                           │
```

1. Агент вызывает `claude -p` (Claude CLI) для принятия решений
2. Claude получает состояние страницы и решает какой инструмент вызвать
3. Playwright выполняет действие в реальном Chromium
4. Результат возвращается Claude → следующее решение
5. Цикл до завершения задачи

## Ключевые особенности

- **Автономность** — агент сам определяет что делать, без захардкоженных шагов
- **Без хардкод-селекторов** — элементы нумеруются динамически через `data-agent-id`
- **14 инструментов** — navigate, click, type, scroll, hover, screenshot и др.
- **Управление контекстом** — автоматическая обрезка длинных сессий
- **Восстановление при ошибках** — агент пробует альтернативные подходы
- **$0 стоимость** — работает через подписку Claude Code, без отдельного API ключа

## Требования

- Node.js 18+
- Claude Code с активной подпиской (`npm i -g @anthropic-ai/claude-code`)
- Playwright + Chromium

## Установка

```bash
git clone <repo-url>
cd browser-agent
npm install
npx playwright install chromium
```

## Запуск

```bash
# Интерактивный режим — вводишь задачи одну за другой
npm start

# Одна задача
npx tsx src/run-task.ts "Найди 3 вакансии AI-инженера на hh.ru"
```

## Технические решения

**Почему Claude CLI вместо API:**
OAuth-токен подписки Claude делит rate limit с Claude Code. Claude CLI (`claude -p`) использует подписку напрямую без конфликтов и без дополнительных затрат.

**Почему Playwright:**
Лучшая поддержка современных сайтов, надёжные локаторы, headed-режим для наблюдения, встроенная обработка диалогов и вкладок.

**Tool calling через JSON в промпте:**
Claude CLI не поддерживает нативный tool use API. Вместо этого агент описывает инструменты в системном промпте и просит модель отвечать JSON-объектами. Парсер извлекает tool calls из ответа, включая код-блоки и inline JSON.

**Динамическая нумерация элементов:**
Вместо CSS-селекторов, BrowserController инжектит `data-agent-id` атрибуты во все интерактивные элементы страницы. Агент видит `[3] <input type="text"> placeholder="Поиск"` и использует номер `3` для взаимодействия.

## Продвинутые паттерны

- **Auto-recovery** — при ошибке клика агент сам пробует альтернативный подход (прямой URL, другой элемент)
- **Context trimming** — при длинных сессиях автоматически обрезает историю, сохраняя системный промпт и последние результаты
- **Nudge mechanism** — если модель отвечает текстом вместо tool call, агент принуждает к действию
