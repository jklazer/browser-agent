# AI Browser Agent

Автономный AI-агент, управляющий реальным браузером для выполнения сложных многошаговых задач.

Пользователь пишет задачу текстом в терминале — агент сам открывает сайты, находит элементы, заполняет формы, кликает и возвращает результат.

## Как работает

```
Пользователь           Agent (Claude CLI)         Playwright (Chromium)
    │                       │                           │
    │── "Найди вакансии" ──>│                           │
    │                       │── navigate(hh.ru) ──────>│
    │                       │<── page state ───────────│
    │                       │── type_text("AI инж") ──>│
    │                       │── press_key("Enter") ───>│
    │                       │── get_page_state ───────>│
    │                       │<── результаты ───────────│
    │<── 3 вакансии ────────│                           │
```

1. Агент вызывает `claude -p` (Claude CLI) для принятия решений
2. Claude получает состояние страницы и решает какой инструмент вызвать
3. Playwright выполняет действие в реальном Chromium
4. Результат возвращается Claude → следующее решение
5. Цикл до завершения задачи

## Ключевые особенности

- **Автономность** — агент сам определяет что делать, без захардкоженных шагов
- **Без хардкод-селекторов** — элементы нумеруются динамически через `data-agent-id`
- **15 инструментов** — navigate, click, type, scroll, hover, screenshot, query_dom и др.
- **Dual mode** — API (`ANTHROPIC_API_KEY`, быстрый, платный) или Claude CLI (подписка, бесплатно)
- **Prompt caching** — 90% скидка на кэшированные system+tools в API mode
- **Safety guards** — code-level блокировка bulk-действий (Select all, >10 чекбоксов подряд)
- **Управление контекстом** — автоматическая обрезка длинных сессий + compaction старых скриншотов

## Требования

- Node.js 18+
- Либо `ANTHROPIC_API_KEY` в `.env` (API mode), либо Claude CLI с активной подпиской
- Playwright + Chromium

## Установка

```bash
git clone <repo-url>
cd browser-agent
npm install
npx playwright install chromium
```

## Запуск

```bash
# Интерактивный режим — вводишь задачи одну за другой
npm start

# Одна задача
npx tsx src/run-task.ts "Найди 3 вакансии AI-инженера на hh.ru"
```

## Технические решения

**Dual mode: API vs CLI:**
Если в `.env` указан `ANTHROPIC_API_KEY` — агент использует нативный tool-use API, быстрее и с prompt caching (90% скидка на повторяющиеся system+tools). Без ключа агент падает на `claude -p` (CLI subscription mode) — медленнее, без нативного tool use (JSON-парсинг в промпте), но бесплатно.

**Почему Playwright:**
Лучшая поддержка современных сайтов, надёжные локаторы, headed-режим для наблюдения, встроенная обработка диалогов и вкладок.

**Tool calling через JSON в промпте:**
Claude CLI не поддерживает нативный tool use API. Вместо этого агент описывает инструменты в системном промпте и просит модель отвечать JSON-объектами. Парсер извлекает tool calls из ответа, включая код-блоки и inline JSON.

**Динамическая нумерация элементов:**
Вместо CSS-селекторов, BrowserController инжектит `data-agent-id` атрибуты во все интерактивные элементы страницы. Агент видит `[3] <input type="text"> placeholder="Поиск"` и использует номер `3` для взаимодействия.

## Продвинутые паттерны

- **Sub-agent architecture** — специализированные промпты по типу задачи (search / email / shopping / profile) определяются автоматически по семантике
- **query_dom** — отдельный Claude-вызов анализирует DOM и отвечает на конкретный вопрос, экономя контекст основной сессии
- **Prompt caching** — system prompt + tools кэшируются (90% скидка при повторных шагах)
- **Security layer** — `ask_user` перед деструктивными действиями; code-level guard блокирует "Select all" и серийные клики по чекбоксам независимо от решения модели
- **Context trimming + screenshot compaction** — старая история обрезается, старые скриншоты заменяются placeholder'ами, оставляя только актуальный
